端口扫描

端口对应网络服务及应用端程序
服务端程序的漏洞通过端口攻入
发现开放的端口
更具体的攻击面

UDP端口扫描

假设 ICMP port-unreachoble 响应代表端口关闭
目标系统不响应ICMP port-unreachoble时，可能产生误判
完整的UDP应用层请求
准确性高
耗时巨大

Scapy UDP Scan

端口关闭：ICMP port-unreachable
端口开放：没有回包
了解每一种基于UDP的应用层包结构很有帮助
与三层相同的技术
误判

Scapy
sr1(IP(dst="1.1.1.1")/UDP(dport=53),timeout=1,verbose=1)

Python ./udp_port_scan.py 1.1.1.1 1 100

NMAP
nmap -sU 1.1.1.1
    默认的1000个参数
    ICMP host-unreachable
nmap 1.1.1.1 -sU -p-
nmap -iL iplist.txt -sU -p 1-200

TCP端口扫描

基于连接在协议
三次握手
隐蔽扫描–syn
只发syn包，返回syn,ack后，不发最后一个ack，不建立完整连接
应用日志不纪录扫描行为，隐蔽
僵尸扫描
极度隐蔽
实施条件苛刻
必须为可伪造源地址
选择僵尸机
闲置系统
系统使用递增的IPID
0
随机
全连接扫描
所有的TCP扫描方式，都是基于三次握手的变化来判断目标端口状态

隐蔽扫描–syn

Syn–syn/ack–rst

Scapy

sr1(IP(dst="192.168.254.254")/TCP(flags="S",dport=22),timeout=1,verbose=0)
sr1(IP(dst="192.168.254.254")/TCP(flags="S"),timeout=1,verbose=1)#默认80端口

python ./tcp_syn_scan.py

Nmap syn半连接隐蔽端口扫描

nmap -sS 1.1.1.1 -p 80,21,25,110,443
nmap -sS 1.1.1.1 -p 1-65535 --open        or    nmap -sS 1.1.1.1 -p- --open
nmap -sS -iL iplist.txt -p 80,21,25,110

hping3 syn半连接隐蔽端口扫描

hping3 1.1.1.1 --scan 80 -S
hping3 1.1.1.1 --scan 80,21,25,443 -S
hping3 1.1.1.1 --scan 0-65535 -S
hping3 -c 10 -S --spoof 1.1.1.1 -p ++1 192.168.254.254 #伪造源地址，欺骗扫描

全连接端口扫描

Syn扫描不需要raw packets
内核认为syn/ack是非法包，直接发rst中断连接
全连接扫描对scapy比较困难
结果最准确

nmap 全连接端口扫描

nmap -sT 1.1.1.1 -p 80
nmap -sT 1.1.1.1 -p 80,21,25
nmap -sT 1.1.1.1 -p 80-2000
nmap -sT -iL iplist.txt  #默认1000个常用端口

dmitry 全连接端口扫描

功能简单，但使用简便
默认150个最常用端口

dmitry -p 1.1.1.1
dmitry -p 1.1.1.1 -o output #输出到文件

nc 全连接端口扫描

nc -nv -w 1 -z 192.168.254.254 1-100
for x in $(seq 20 30); do nc -nv -w 1 -z 1.1.1.1 $x; done | grep open
for xin $(seq 1 254); do nc -nv -w 1 -z 1.1.1.$x 80; done

Scapy
Python ./zombie.py

nmap 僵尸扫描

发现僵尸机

nmap -p445 192.168.254.254 --script=ipidseq.nse

扫描目标

nmap 1.1.1.1 -sl 1.1.1.2 -Pn -p 0-100
